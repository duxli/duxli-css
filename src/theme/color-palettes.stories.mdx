# Color Palettes

With Duxli, colors are named by intent so it is easy to determine how they're used in the theme.

Color palettes include the following shades:

| Category | Named Shade | Secondary Shades          | Description                                                                               |
| -------- | ----------- | ------------------------- | ----------------------------------------------------------------------------------------- |
| Main     | `*-main`    | `*-400`, `*-500`, `*-600` | Main shades for the color. These must all contrast with `*-contrast`.                     |
| Light    | `*-light`   | `*-100`, `*-200`, `*-300` | Lightest shades of the color. These should always contrast with `--dx-color-neutral-999`. |
| Dark     | `*-dark`    | `*-700`, `*-800`, `*-900` | Darkest shades of the color. These should always contrast with `--dx-color-neutral-000`.  |

Lighter colors, like yellow, will likely have lighter main shades with a dark value for `*-contrast`.
Darker colors, like navy, will likely have darker main shades and a light `*-contrast`.
Either option is fine, so long as all shades in the main category have sufficient contrast against the specified contrast color for the palette.

This pattern limits the shades that can be used, but is predictable and simple.

## Neutrals

The neutral palette is used across the theme for backgrounds, text, borders, and more.

Neutral colors have additional color shades in the light and dark categories.

The neutral palette also includes `--dx-color-neutral-000` and `--dx-color-neutral-999`.
These shades represent the lightest and darkest colors for the app.
Effectively, these are black and white, but any color can be used.
